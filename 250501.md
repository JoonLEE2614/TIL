## 배운거

스케일링 기법
Min-Max 스케일링:

목표: 변수의 값을 0과 1 사이로 변환.
특징: 데이터의 최소값과 최대값을 기준으로 상대적인 위치를 정함.
단점: 이상치에 민감하여, 이상치가 있을 경우 변환된 값이 왜곡될 수 있음.

표준화 (Standardization):

목표: 변수의 평균을 0, 표준편차를 1로 변환하여 정규분포 형태로 변환.
특징: 평균과 표준편차를 기준으로 수치가 변환되어, 이상치의 영향이 비교적 적음.
장점: 거리 기반 알고리즘에 유리.

로버스트 스케일링 (Robust Scaling):

목표: 중앙값과 IQR(Interquartile Range)을 기준으로 변환.
특징: 이상치에 강건한 효과.
장점: 이상치가 포함된 데이터에 적합.

구간화 (Binning, Bucketing):

목표: 연속형 수치를 범주형으로 변환.
특징: 데이터의 범위나 빈도를 기준으로 구간을 나눔.
장점: 데이터의 복잡도를 줄이고, 이상치를 완화시킬 수 있음.
단점: 수치 정보의 손실이 있을 수 있음.

범주형 변수 다루기
원-핫 인코딩 (One-Hot Encoding):

목표: 범주형 변수를 이진 벡터로 변환.
장점: 범주가 크면 알고리즘에서 변수의 의미를 정확히 파악할 수 있음.
단점: 범주가 많아질 경우 차원의 저주 발생, 메모리 및 연산에 부담.

레이블 인코딩 (Label Encoding):

목표: 범주를 숫자로 변환.
장점: 간단하게 변환 가능하며, 메모리 관리에 유리.
단점: 순서가 없는 범주를 숫자 순서대로 변환해 의미를 왜곡할 수 있음.

빈도 인코딩 (Frequency Encoding):

목표: 범주의 빈도값을 숫자로 변환.
장점: 빈도 정보를 학습에 활용 가능.
단점: 범주의 빈도가 같으면 의미를 왜곡할 수 있음.

타겟 인코딩 (Target Encoding):

목표: 타겟 변수에 따른 범주형 변수를 수치로 변환.
장점: 범주 간 수치적인 관계를 반영할 수 있음.
단점: 과적합의 위험, 데이터 누수 문제 발생 가능.

타겟 인코딩의 과적합 방지 방법

스무딩 (Smoothing):

목표: 데이터셋의 평균값을 기준으로 타겟 인코딩된 값을 조정하여 과적합을 방지.

K-Fold 교차 검증:

목표: 데이터를 여러 번 나누어 타겟 인코딩을 적용해 과적합을 방지하고 다양한 인코딩 값을 생성.
